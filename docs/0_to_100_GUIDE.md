# 🚀 네이버 최소가 추적 자동화 시스템 구축 가이드 (비개발자용)

본 가이드는 프로그래밍 지식이 없는 담당자도 처음부터 끝까지 독립적으로 **가격 추적 자동화 시스템을 구축하고 배포**할 수 있도록 작성된 단계별 매뉴얼입니다. 안내된 순서에 따라 설정을 진행해 주시기 바랍니다.

---

## 🔍 시작하기 전에: 필수 플랫폼 2가지 안내
본 시스템이 24시간 동안 자동으로 실행되고 데이터를 축적하기 위해 다음 두 가지 무료 플랫폼을 활용합니다.

1. **GitHub (깃허브)**: 소스 코드가 저장되는 공간이자, 정해진 시간마다 자동으로 가격을 수집하는 스케줄러(Actions)가 구동되는 메인 플랫폼입니다. 또한, 수집된 데이터를 시각화하는 대시보드 웹사이트를 호스팅합니다.
2. **GCS (Google Cloud Storage)**: 매일 수집되는 가격 및 상품 데이터를 영구적으로 안전하게 보관하는 클라우드 저장소입니다. 이 환경이 설정되어야만 장기간의 데이터 누적 및 추세 확인이 가능합니다.

## STEP 1. 프로젝트 복제 (Fork)
관리할 가격 추적기의 원본 프로젝트를 담당자 본인의 GitHub 계정으로 복제(Fork)하여 개인 운영 환경을 구성합니다.

1. [GitHub](https://github.com/) 에 가입 후 로그인합니다.
2. 본 프로젝트 저장소(Repository) 우측 상단에 위치한 **Fork** 아이콘을 클릭합니다.
3. `Create a new fork` 화면 하단의 초록색 **Create fork** 버튼을 클릭합니다.
4. 생성이 완료되면 인터넷 주소창의 URL이 `https://github.com/본인계정명/price-tracker-configuration-ver` 형식으로 변경되었는지 확인합니다.

## STEP 2. 연동용 API 키 및 인증 정보 발급
시스템이 외부 서비스(네이버 검색, 구글 클라우드, 이메일 시스템)와 정상적으로 통신하기 위한 접근 권한 및 인증 키를 발급받습니다. 발급 받은 총 3가지 정보는 메모장에 일괄 저장해 주시기 바랍니다.

### 🔑 2-1. 네이버 API 키 발급 (쇼핑 검색 데이터 수집 권한)
1. [네이버 개발자 센터](https://developers.naver.com/apps/#/register) 에 접속하여 로그인합니다.
2. 애플리케이션 등록 화면에서 아래와 같이 입력합니다.
   - 애플리케이션 이름: 임의 지정 (예: `기획팀 가격 추적 시스템`)
   - 사용 API: **검색** 선택
   - 비로그인 오픈 API 서비스 환경: **WEB 설정** 선택 후 URL 란에 `http://localhost` 입력
3. 하단의 **등록하기**를 클릭하면 완료 화면에서 **Client ID** 와 **Client Secret** 이 표시됩니다. 
4. 두 가지 문자열 내역을 메모장에 복사해 둡니다.

### 🔑 2-2. Google Cloud Storage 키 발급 (누적 데이터 보관 권한)
수집된 가격 데이터를 영구적으로 저장하기 위한 버킷과 시스템 접근 권한을 생성합니다.
1. [Google Cloud Console](https://console.cloud.google.com) 에 접속 후 새 프로젝트를 하나 생성합니다.
2. 좌측 메뉴에서 **Cloud Storage** → **버킷(Bucket)** 으로 이동 후 **"만들기"**를 클릭합니다.
   - 버킷 이름은 글로벌 환경에서 중복되지 않는 이름으로 유일해야 합니다. (예: `company-price-tracker-1234`) → **설정한 버킷 이름을 메모합니다.**
   - 리전(위치)은 속도가 빠른 `asia-northeast3` (서울) 로 지정하고, 나머지 설정은 기본값으로 둔 상테로 생성을 완료합니다.
3. 자동화 시스템이 이 저장소에 데이터를 적재할 수 있도록 서비스 계정을 생성합니다. 상단 ≡ 메뉴 → **IAM 및 관리자** → **서비스 계정**을 클릭합니다.
   - "서비스 계정 만들기" 클릭 후 임의의 계정명을 지정하고 '다음'을 누릅니다.
   - **(주의)** 역할 선택 항목에서 반드시 **Storage 개체 관리자** 자격을 부여한 뒤 완료해야 합니다. (조회용 권한은 불가합니다)
4. 생성된 서비스 계정을 클릭 → 상단 **'키'** 탭 선택 → **'키 추가'** → **'새 키 만들기'**를 진행합니다.
5. 키 유형을 **JSON**으로 선택하고 만들기 누르면 설정 파일이 로컬 PC 다운로드 폴더에 자동 다운로드됩니다.
   - 해당 JSON 파일을 편집장(메모장 등)으로 열어, 양 쪽 끝의 `{ }` 괄호를 포함한 전체 내용 소스를 메모장에 함께 복사해 둡니다.

### 🔑 2-3. Gmail 앱 비밀번호 발급 (알림 이메일 발송 권한)
목표 상품의 가격 변동 내역이 감지될 시 이메일을 발송하기 위해 사용할 Google 계정의 전용 암호를 생성합니다.
1. 관리용 구글 계정 접속 → 우측 상단의 계정 프로필 → **Google 계정 관리** → 좌측의 **보안** 탭으로 이동합니다.
2. **2단계 인증** 항목이 '사용'으로 활성화되어 있는지 확인합니다. (비활성화 상태라면 휴대전화 인증을 거쳐 활성화 처리합니다.)
3. 상단 돋보기 아이콘의 검색창에 **앱 비밀번호**를 검색하여 진입합니다. (앱 이름은 기기를 식별할 수 있는 수준으로 자유롭게 기재합니다.)
4. 화면에 생성된 **16자리 영문자 비밀번호** 텍스트 블록 전체를 복사하여 띄어쓰기 없이 메모장에 저장해 둡니다.

## STEP 3. 프로젝트 내 환경 변수(Secrets) 및 권한 등록
STEP 2에서 사전 수집된 인증 정보들을 GitHub 프로젝트 내 안전한 가상 저장소(Secrets)에 등록합니다.

1. 복제한 레포지토리(Repository) 메인 화면 상단의 ⚙️ **Settings** 메뉴를 클릭합니다.
2. 좌측 패널 하단의 **Secrets and variables** → **Actions** 항목을 클릭합니다.
3. 우측의 초록색 **New repository secret** 버튼을 클릭하여, 하단 표에 기재된 6개의 항목별 Name과 Secret 값을 대응하여 신규 등록합니다.

| Name (변수명) | Secret (등록할 값) | 발급 출처 (참고 단계) |
|---|---|---|
| `NAVER_CLIENT_ID` | 발급받은 영문/숫자 혼합 ID | **STEP 2-1** |
| `NAVER_CLIENT_SECRET` | 발급받은 영문/숫자 혼합 Secret | **STEP 2-1** |
| `GCS_BUCKET` | 생성한 스토리지 통(버킷) 명칭 | **STEP 2-2** |
| `GCP_SA_KEY` | 다운로드한 JSON 파일 내부 내용 전체 | **STEP 2-2** |
| `EMAIL_FROM` | 발송 주체로 사용할 소유자 Gmail 주소 | (사용자 계정) |
| `EMAIL_APP_PASSWORD` | 시스템에서 발급 받은 16자리 앱 전용 비밀번호 | **STEP 2-3** |

> **선택사항:** 메일 수신 대상자를 팀원 전체로 다수 지정하시거나, 발신용 메일이 아닌 별도의 타사 업무용 메일로 알림 수신을 원하실 경우 Name 필드에 `EMAIL_TO` 를 추가하시고 Secret 블록 상에 수신할 대상 이메일 주소를 쉼표(,) 간격 구분으로 병기해 주시기 바랍니다.

## STEP 4. 자동 수집 및 대시보드 활성화 (Actions & Pages)
운영에 필요한 환경 세팅이 완료되었습니다. 자동 수집 프로세스 구동 및 데이터 시각화 웹페이지를 활성화시킵니다.

### 4-1. 일일 자동 스케줄러 실행 (GitHub Actions)
1. 레포지토리 상단의 **▶️ Actions** 탭을 클릭하여 이동합니다.
2. 안내 화면 상에 나타나는 **"I understand my workflows, go ahead and enable them"** 동의 버튼을 클릭합니다.
3. 좌측 워크플로우(Workflow) 목록 영역에서 **Naver Price Tracker (Automated)** 타이틀을 클릭합니다.
4. 우측 컨텍스트 메뉴인 **Run workflow** ▾ 드롭다운을 열고, 내부에 위치한 **Run workflow** 버튼을 실행합니다.
   - 최초 시스템 구동 테스트가 개시됩니다. 로딩 아이콘이 종료되고 녹색 ✅ 체크 시그널이 발생할 시 시스템이 정상 적재 및 작동함을 의미합니다.

### 4-2. 데이터 대시보드 (GitHub Pages) 연동
수집된 일자별 데이터를 브라우저에서 열람 가능한 웹사이트 UI로 포팅합니다.
1. 상단 컨트롤 패널에서 ⚙️ **Settings** 로 재이동합니다.
2. 좌측 리스트업 중 **Pages** 항목을 클릭합니다.
3. 페이지 가운데 **Build and deployment** 항목의 **Source** 옵션 값을 기본형에서 **GitHub Actions** 항목으로 변경합니다.
4. 프로세싱까지 기준 5~10분이 소요됩니다. 이후 리퍼시(새로고침) 시, 상단 영역에 고유 대시보드용 URL 주소(`https://계정명.github.io/프로젝트명`)가 형성됩니다. 해당 링크를 업무용으로 저장/북마크하여 관리하시면 됩니다.

## STEP 5. 추적 대상 상품 인덱스 구성 (with 생성형 AI)
마지막 절차로 시스템이 실질적으로 추적 검토해야 할 '목표 상품 목록(`targets.yaml`)'을 세팅합니다. YAML 코드를 수기 작성하는 공수 리스크를 최소화하기 위해 사내에서 이용 가능한 생성형 AI(ChatGPT, Claude 등)를 사용해 지시어를 도출합니다.

1. 아래 기재된 '프롬프트 입력 양식' 블록의 텍스트 내용을 전체 복사합니다.
2. `[예시]`로 대괄호 및 예문이 들어간 데이터 규격을 현재 실무상 요구되는 기획 모델 정보로 갱신합니다.
3. 양식이 보정된 전체 프롬프트를 AI 챗봇에 입력하여 YAML 형식의 결괏값을 생성받습니다.

```text
아래 정보를 바탕으로 네이버 쇼핑 가격 추적기 환경의 targets.yaml 소스 파일을 생성해줘.

=== 타겟팅할 목표 상품 정보 정보 ===
[상품 1]
- 상품명: [예: LG 오브제컬렉션 스타일러]
- 검색 키워드: [예: 엘지 오브제컬렉션 스타일러 화이트]
- 순위 검색어: [예: LG 스타일러]
- 카테고리: [예: 생활가전, 홈 케어 등]
- 네이버 쇼핑 product_id: [예: 53507707536] (네이버 쇼핑 상품 상세 페이지 URL 최하단 숫자)
- 반드시 포함되어야 할 고정 키워드: [예: 화이트, 오브제]
- 제외시킬 필터링 키워드: [예: 중고, 리퍼, 케이스, 렌탈]

[상품 2] (추가 모니터링이 필요한 상품이 존재할 경우 하단에 동일 양식으로 리스트업)
...

=== 시스템 출력 규칙 (준수 필수) ===
- product_id 항목 수치는 반드시 양 끝 구성을 따옴표('')로 감쌀 것
- url 파라미터의 마지막 숫자는 product_id와 동일 수치로 매칭할 것
- 에러 방지를 위해 탭(Tab) 사용 기호는 차단하며, 스페이스(Space) 2번을 통해 들여쓰기 위계를 준수할 것
- browser 섹션에 소속된 기본 항목(wait_until, price_selector 등)을 누락 발생 없이 현상유지 체계로 복원할 것
- 출력 파일의 최상단 열에 common: (exclude, display 50, timeout 20) 섹션을 전역 변수로 1회만 단독 배치할 것
- 사용자 적용 복지가 원활히 진행될 수 있도록 다른 설명 생략하에 오로지 YAML 소스 코드만 블록 포맷 형태로 렌더링해 줄 것.
```

### 🎯 추출된 설정 파일(`targets.yaml`) 메인 서버 적용 및 리로드
1. 레포지토리의 기본 첫화면(좌측 최상단 <> Code 탭)으로 진입하여 목록 중에서 **`targets.yaml`** 리소스를 클릭해 진입합니다.
2. 뷰어 화면 우측에 존재하는 연필 모양의 ✏️ **Edit(수정)** 아이콘을 엽니다.
3. 화면상 기재되어 테스트 중이던 기본 코드를 모두 텍스트 에리어에서 삭제 후, 직전 과정에서 AI가 조작해준 생성 코드 텍스트를 위상 변경 없이 **그대로 붙여넣기(Paste)** 하십시오.
4. 우측 상단의 녹색 **Commit changes...** 버튼을 클릭하신 뒤, 출력되는 컨펌 창에서 한 차례 더 저장 승인 처리를 이행합니다.
*(본 과정 처리 직후 시스템 이벤트가 유발되어, 업데이트된 최종 상품 리스트를 기반으로 데이터를 순차 신규 로드하기 시작합니다.)*

## 🛠 주요 트러블슈팅 및 장애 대처(FAQ)

> **Q 1. 구동 메뉴(Actions) 클릭 시, 실패(Red Cross) 사인이 나타나고 프로세스가 정지됩니다.**  
> **A.** 대다수의 이슈 케이스는 **STEP 3**의 환경 변수(Secrets) 세팅 과정 시 원천 데이터의 값이 누락되거나, 또는 문자열 말미에 띄어쓰기(공백) 혹은 개행 오류가 등록되어 파싱(분석)이 실패되는 증상으로 확인됩니다. Settings -> Secrets 메뉴 접근 후 저장된 변수항들을 일괄 철회 삭제하시고, 로컬 메모장에서 불순물이 혼합되지 않았음을 체크하신 후 재구성 입력을 부탁드립니다.

> **Q 2. 소스코드에 입력할 대상 품목 고유 번호(`product_id`)는 어디서 확보해야 합니까?**  
> **A.** 일반 브라우저로 네이버 쇼핑 접속 후 추적을 실시할 목적 상품의 상세 카탈로그 페이지에 액세스 하신 뒤, 상단의 브라우저 주소 검색창 표출 내용을 스크림(`.../catalog/1234567890`)해보시면 최후단 구문 내 일련의 넘버링 정보 값을 확인하실 수 있습니다.

> **Q 3. 설정은 매뉴얼에 따라 모두 이식 완료하였으나 가격 변동 알림 이메일 핑이 들어오지 않습니다.**  
> **A.** 발송 주체용 Google 측 계정이 속한 보안 관리 레벨상 계정 2단계 인증 프로텍트가 미비되어 있거나, 상단 STEP 2-3 메뉴 기반으로 발행된 별도의 '16자리 앱 비밀번호' 대신 기존 평문 비밀번호를 환경 변수로 강제 세팅했을 경우가 주된 부적합 사유입니다. 정책상 서드파티 접근은 별도의 고립된 앱 전용 비밀번호가 필수로 가용되어야 합니다.
